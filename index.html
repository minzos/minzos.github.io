MinzOS

MinzOS is not z/OS

An operating system to support high performance, COBOL based applications.

The objective is to build a GNU/Linux based operating system which would be capable of high performance, resilience, security and ease of administration.

  • Should be 64-bit allowing support for upto 248 bytes of memory under x86-64 based hardware.
  • Should be capable of maintaining data integrity at all times.
  • Should be capable of running lots of tasks in soft-realtime.
  • Should be highly responsive.
  • Should be capable of recovering from a disaster quickly.
  • Should be easy to administer.
  • Should be available for use at all times.
  • Should be capable of tracking all operations performed on the system in the past.
  • Should not be easy to compromise.

Typical usage scenario would involve a minimum of 1,000 thin clients (in pure-console mode) connecting to the system concurrently over secure TCP/IP connections to perform operations for creating, updating, deleting and printing records.

Thin clients could be simple Raspberry Pi based devices connected to a network and possessing a display, a keyboard and a dot-matrix printer as it's peripherals, running a customized linux distribution which would allow logging into the system via a username + password scheme.

The main system itself should be administerable only via a root login at site.

Users of the system should be allowed to access the system only via a set of menu driven commands, which if by-passed, should prevent the user from roaming around through the root of the filesystem.
